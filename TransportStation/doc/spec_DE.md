Globale Beschreibung: **Das Datenmodell ist eine allgemeine Beschreibung von städtischen Stationen (Metro, Bus, Straßenbahn, Heliport, ...) nach dem GFTS-Standard https://developers.google.com/transit/gtfs/reference/#stopstxt, sowie die detaillierte Beschreibung dieser (Zugangsmittel, Bahnsteig, Hilfestellung, ...).**  

## Liste der Eigenschaften  

- `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `contactPoint`: Die Angaben zur Kontaktaufnahme mit dem Artikel.  - `contractingAuthority`: Name des öffentlichen Auftraggebers.  - `contractingCompany`: Name des Vertragsunternehmens, das für den Betrieb der Station verantwortlich ist.  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateLastReported`: Ein Zeitstempel, der den letzten Zeitpunkt angibt, an dem das Gerät erfolgreich Daten gemeldet hat. Datum und Uhrzeit in einem ISO8601 UTC-Format.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `dimension`: Globale Dimension. Das Format wird durch eine Untereigenschaft von 3 Elementen strukturiert. Der Einheitencode (Text) wird unter Verwendung der [UN/CEFACT Common Codes] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes) angegeben. Zum Beispiel steht **MTR** für Meter  - `id`: Eindeutiger Bezeichner der Entität  - `installationMode`: Standort relativ zur Bodenreferenz. Enum:'aerial, ground, underGround, underSea'  - `inventory`: Allgemeines Datenmapping nur für `locationType` = 0, 1, 3, 4. Das Format wird durch eine Untereigenschaft von 4 Elementen strukturiert.  - `levelId`: Etage, auf der sich der Standort befindet. Numerischer Index, der mit der Etage verbunden ist. Gibt die relative Position dieser Etage im Verhältnis zu den anderen an. Der Index 0 zeigt das Erdgeschoss an. Die Etagen oberhalb des Erdgeschosses werden durch positive Indizes und die unterirdischen Etagen durch negative Indizes angezeigt.  - `location`:   - `locationType `: Link zum GTFS-Standard-Repository, das den unterschiedlichen Ort beschreibt [Ortstyp]. 0 Haltestelle oder Bahnsteig (Ort, an dem Benutzer in ein öffentliches Verkehrsmittel ein- oder aussteigen). 1 Station (Bereich oder physische Struktur, die einen oder mehrere Bahnsteige umfasst). 2 Eingang oder Ausgang (Ort, an dem Benutzer eine Station von der Straße aus betreten/verlassen können). 3 Generische Kreuzung (Ort in einer Station, der keinem anderen `location_type`-Wert entspricht). 4 Einstiegsbereich eines bestimmten Ortes auf einem Bahnsteig, an dem Benutzer in ein Fahrzeug ein- / aussteigen können.  - `name`: Der Name dieses Elements.  - `openingHoursSpecification`: Ein strukturierter Wert, der Informationen über die Öffnungszeiten eines Ortes oder einen bestimmten Dienst innerhalb eines Ortes liefert  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `parentStation`: Link zum GTFS-Standard-Repository, der die unterschiedliche Verbindung zwischen Station und Plattform beschreibt [Parent STATION]. Fall '1' location_type = 0 (Haltestelle / Bahnsteig ), das Feld parent_station enthält die ID einer Station. Fall '2' location_type = 1 (Station), dieses Feld muss leer sein. Fall '3' location_type = 2 (Eingang / Ausgang) oder location_type = 3 (generische Kreuzung), das parent_station-Feld enthält die ID einer Station location_type = 1. Fall '4' location_type = 4 (Einstiegsbereich), das parent_station-Feld enthält die ID eines Bahnsteigs.  - `platformCode`: Bahnsteigbezeichner für eine Haltestelle vom Bahnsteig-Typ `location_type` = 0, wenn die Haltestelle in einem Bahnhof liegt.  - `refPointOfInterest`: Ein Verweis auf ein Sonderziel, das mit dieser Beobachtung verbunden ist.  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `stationConnected`: Von dieser Station aus mögliche Verbindungen. Ein strukturierter Wert von 0 bis N Vorkommen, wobei jedes Element eine Zeichenkette im Format `stationType` : [Liste der angeschlossenen Linien, getrennt durch ein Komma] ist. Enum:'aerialLift, bus, cableTram, ferry, funicular, monorail, rail, subway, train, tram, trolleybus'  - `stationType`: Art der Transportstation. Enum:'aerialLift, bus, cableTram, ferry, funicular, monorail, rail, subway, trolleybus, tram'  - `type`: NGSI Entity-Typ. Es muss TransportStation sein  - `webSite`: Link zur offiziellen Website für weitere Informationen...  - `wheelChairAccessible `: Zugang für Person mit eingeschränkter Mobilität möglich. Für Haltestellen ohne Eltern 0 sind keine Informationen über die Zugänglichkeit der Haltestelle verfügbar. 1 einige Fahrzeuge an dieser Haltestelle können einen PMR-Benutzer einsteigen. 2 PRM-Benutzer kann an dieser Haltestelle nicht einsteigen. Für eine Haltestelle, die Teil einer Station ist 0 erbt die Haltestelle das wheelchair_boarding Verhalten der übergeordneten Station, wenn diese ausgefüllt ist. 1 Fahrspuren bieten Rollstuhlzugang zu der Haltestelle / dem Bahnsteig von außerhalb der Station. 2 keine Fahrspur bietet Rollstuhlzugang zur Haltestelle / zum Bahnsteig von außerhalb der Station. Für Stationseingänge / -ausgänge 0 der Stationseingang erbt das wheelchair_boarding-Verhalten der Hauptstation, falls angegeben. 1 der Stationseingang ist rollstuhlgerecht. 2 kein rollstuhlgerechter Weg verbindet den Stationseingang mit den Haltestellen / Bahnsteigen.  - `zoneId`: Preiszone der Station.
